1. Xác định mục đích: 
	1.1 Quản lý thư viện sách
	1.2 Quản lý tài nguyên (hình ảnh, tài liệu, files)
	1.3 Phân quyền user (khách/thành viên/Editor)

2. Lên ý tưởng hệ thống web: 
 (Dựa trên mục đích, lên ý tưởng hệ thống web)
	2.1 Phân tích nhu cầu sử dụng để lên ý tưởng giao diện menu sao cho tâm lý nhất, gọn gàng khoa học, dễ dùng nhất.
	2.2 Dựa trên giao diện menu, xác định cấu trúc hệ thống nội dung cho tối ưu nhất.
	2.3 Dựa trên hệ thống nội dung, lên ý tưởng cấu trúc cơ sở dữ liệu.

3. Xác định yêu cầu kỹ thuật 
 (Dựa trên ý tưởng hệ thống web, và quy mô dự án, xác định các công nghệ cần dùng)
	3.1 Công nghệ phần mềm: 
	  + Ngôn ngữ: HTML/CSS/JS/PHP/SQL
	  + Nền tảng: dùng công nghệ web để sử dụng trên đa nền tảng (Smartphone/Máy tính)
	  + Webserver: chạy trực tiếp bằng lệnh của php
	  + Quy mô: Network nội bộ (các thiết bị trong mạng LAN cùng truy cập web qua IP LAN của một máy chủ chạy php)
	3.2 Công nghệ phần cứng và thiết bị: 
	  + Máy tính chạy hệ điều hành bất kỳ 
	    (miễn chạy được php-cli),
	    RAM tối thiểu 8GB để chịu tải số lượng lớn truy cập
	    Bộ nhớ lưu trữ (ổ cứng) tối thiểu 20GB để chứa các tài liệu/tài nguyên đủ cho quy mô dự án
	  + Router mạng đủ mạnh để cân bằng tải nhiều thiết bị truy cập

4. Xây dựng website 
	4.1 Cài đặt webserver: php, sqlite
	4.2 Dựa trên ý tưởng hệ thống, xây dựng Front-End 
	    (giao diện, menu, các tương tác ví dụ tìm kiếm, hộp thoại, nút bấm,..)
	4.3 Dựa trên ý tưởng hệ thống, xây dựng Back-End: 
	    chương trình php để xử lý các yêu cầu từ Front-End, các tương tác với dữ liệu (tìm/thêm/sửa/xóa)

5. Nhập dữ liệu cho web

	7.2 Triển khai máy chủ và web cho chạy trên hệ thống LAN
	7.3 Xem xét thảo luận về việc triển khai web ra internet (hosting/domain)
